◆ redshow_analyze()

EXTERNC redshow_result_t redshow_analyze ( uint32_t  thread_id,
uint32_t  cubin_id,
uint64_t  kernel_id,
uint64_t  host_op_id,
gpu_patch_buffer_t *  trace_data 
)

Apply registered analysis to a gpu trace, analysis results are buffered. redshow_callback_func is called when the analysis is done. Multi-threading is enable by export OMP_NUM_THREADS=N.

First use binary search to find an enclosed region of function addresses instruction_offset = instruction_pc - function_address

Parameters
thread_idWhich thread launches the kernel
cubin_idLookup correponding cubin
kernel_idUnique identifier for a calling context
host_op_idUnique identifier for the operation
trace_dataGPU memory trace for a single kernel launch.
Returns
EXTERNC

@thread-safe yes
